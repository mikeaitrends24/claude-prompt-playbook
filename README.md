## 🎁 Free Samples (from Chapter 1: Code & Debugging)

### Prompt 1: The Bug Finder & Explainer
**When to use:** You have code that's broken and you don't know why.
You are a senior software engineer with 15 years of experience in [LANGUAGE/FRAMEWORK].
Here is my code:
[PASTE YOUR CODE]
Please:

Find every bug — including ones that aren't causing errors yet
Explain WHY each bug is wrong (not just what to change)
Show me the corrected code with comments explaining each fix
Tell me what I should have been thinking to avoid this bug

Be specific. Don't just say "this is wrong" — tell me the exact mental model I'm missing.
**What to expect:** A line-by-line breakdown of every issue with corrected code.

### Prompt 7: The Error Message Decoder
**When to use:** You see an error message you don't understand.
I'm getting this error message and I don't understand it:
[PASTE YOUR EXACT ERROR MESSAGE]
My code:
[PASTE RELEVANT CODE]
My setup:

Language/Framework: [NAME AND VERSION]
Operating System: [OS]
What I was doing when this happened: [DESCRIBE]

Please:

Explain what this error means in plain English
Tell me the most likely cause in my specific code
Give me the step-by-step fix
Tell me how to prevent this error in the future
If there are multiple possible causes, list them in order of likelihood


**What to expect:** A clear explanation and a specific fix. Saves most developers 20+ minutes per error.

### Prompt 13: The Security Vulnerability Checker
**When to use:** Before shipping any code that handles user data, authentication, or payments.
You are a cybersecurity expert specializing in [WEB/MOBILE/API] security.
Review this code for security vulnerabilities:
[PASTE YOUR CODE]
Check specifically for:

Injection attacks (SQL, XSS, command injection)
Authentication and authorization issues
Sensitive data exposure (passwords, tokens, PII)
Insecure dependencies or functions
Missing input validation
Improper error handling that leaks information
Any OWASP Top 10 vulnerabilities

For each vulnerability found:

Severity: Critical/High/Medium/Low
What an attacker could do with it
The exact fix

**What to expect:** A security audit with severity ratings and specific fixes.
